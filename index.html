统计学基础

🎯学完本节你会掌握：描述统计核心指标；假设检验框架（p值/α/I类II类错误）；常用检验类型选择。
📚前置知识：无，本节是起点。学完后推荐：A/B 测试 → 机器学习概览。
💡 直觉类比：把统计检验想成"法庭审判"
H₀（原假设）= 无罪推定：我们默认"没有效果/没有差异"，除非有充分证据。
p-value = 巧合概率：若被告真的无辜，出现这种证据的概率有多低？p = 0.03 意味着"如果真没差异，出现这个结果只有 3% 的概率"——够低，就说'证据确凿，拒绝无辜假设'。
α = 判决门槛：设 α=0.05 就是说"我能接受 5% 的概率冤枉无辜的人（第一类错误）"。
统计功效（Power）= 真凶不漏网的能力：Power=0.8 意味着"真有差异时，我有 80% 的概率发现它"。

描述统计核心指标

集中趋势

均值 / 中位数 / 众数

均值受异常值影响大；收入类数据用中位数；类别数据用众数

离散程度

方差 / 标准差 / IQR

IQR = Q3−Q1，用于检测异常值（1.5×IQR 规则）；CV = σ/μ 用于比较不同量纲

分布形态

偏度 / 峰度

正偏（右偏）：均值>中位数>众数；收入、销量常见；对数变换可处理

假设检验框架

假设检验是判断"观察到的差异是真实差异还是随机噪声"的统计工具。

H₀（原假设）：没有效果 / 没有差异 H₁（备择假设）：有效果 / 有差异 α（显著性水平）：通常取 0.05，即允许 5% 的第一类错误 p-value：在 H₀ 为真时，观察到当前数据（或更极端）的概率 结论：p < α → 拒绝 H₀（统计显著）
⚠️常见误解：p=0.03 不代表"B比A好的概率是97%"，只代表"如果没有差异，随机出现当前结果的概率为3%"。统计显著 ≠ 实际重要（需要 Effect Size）。
检验类型适用场景前提
双样本 t 检验比较两组均值（A/B测试）正态分布，方差齐性
配对 t 检验同一用户前后对比差值服从正态分布
卡方检验比较比例、分类数据独立性期望频数 ≥ 5
Mann-Whitney U非正态分布的两组比较非参数，无分布假设
Z 检验大样本比例检验n≥30，np≥5

置信区间（Confidence Interval）

Confidence Interval CI = x̄ ± zα/2 · σ/√n　　（σ已知，大样本） CI = x̄ ± tα/2, n-1 · s/√n　　（σ未知，小样本，用 t 分布） 直觉：95% CI 意味着"用同样的方法重复抽样100次， 约95次得到的CI会包含真实总体均值" ≠ "有95%的概率真实均值在这个区间"（这是贝叶斯解释）

统计检验力（Statistical Power）

Power Analysis 检验力 Power = 1 − β = P(拒绝H₀ | H₁为真) β = 第二类错误（漏报）概率；通常要求 Power ≥ 0.8 影响Power的四个因素（调一个会影响其他三个）： ↑ 样本量 n → Power ↑ ↑ 效应量 δ → Power ↑（真实差异越大越容易检测） ↑ 显著性 α → Power ↑（但I类错误也增加） ↑ 总体方差 σ² → Power ↓（噪声越大越难检测） 样本量公式：n = 2(z_α/2 + z_β)² σ² / δ²

中心极限定理（CLT）

Central Limit Theorem 无论总体分布如何，当样本量 n 足够大（通常 ≥ 30）， 样本均值 X̄ 的抽样分布趋近于正态分布： X̄ ~ N(μ, σ²/n) 重要性：这是几乎所有参数检验（t检验/Z检验）的理论基础。 即使原始数据非正态（如收入、点击量）， 均值的分布也是近似正态的 → 可用 t 检验。

💡 贝叶斯 vs 频率学派的根本分歧

频率学派：参数是固定未知量，数据是随机的。p-value/CI 基于"无限次重复实验"。

贝叶斯学派：参数本身有概率分布（先验），结合数据得到后验。P(H₀|data) 有直接意义。

工业界 A/B 测试主流是频率学派；贝叶斯 A/B 测试（如 Optimizely）允许"随时停止"但需设计先验。

❌ 统计学最常见的 3 个误解
  • 误解1："p=0.03 说明有97%的概率B更好" — 错！p-value 是 P(数据|H₀)，不是 P(H₀|数据)。它只是说"如果没有差异，出现这个结果的概率是3%"。
  • 误解2："95% CI 有95%的概率包含真值" — 错！频率学派的 CI 含义是：如果重复实验 100 次，约 95 次的 CI 会包含真值。对当前这个 CI，真值要么在要么不在，没有概率。
  • 误解3："统计显著 = 实际重要" — 错！有了百万用户的样本，就算只有 0.001% 的微小差异也会显著。必须配合效应量（Cohen's d、Relative Lift）判断业务价值。

A/B 测试

🎯学完本节你会掌握：A/B 测试完整设计流程；样本量计算；常见陷阱（peeking/novelty effect/网络效应）；结果解读。
📚前置知识：统计学基础（假设检验/p值）。学完后推荐：机器学习概览 → 特征工程。
✅A/B 测试是产品数据科学家最高频的工作场景。面试中要能从头设计一个 A/B 测试流程。
💡 直觉类比：A/B 测试就是"临床试验"在互联网的翻版
新药临床试验怎么做？随机分两组，一组吃药（实验组），一组吃安慰剂（对照组），盲测，跑满足够长时间，最后统计检验。A/B 测试完全一样：
• 随机分流 = 随机分配患者，消除个体差异（高活用户不能都在实验组）
• AA 测试 = 给两组都吃安慰剂，验证随机分组是否均匀（若 AA 测试就显著，说明分流有问题）
• 不能 Peeking = 不能"中期看一眼就停止"，就像临床试验不能因为前10个病人好转就宣布新药有效
• 跑满一个完整周期 = 至少一周（去除星期效应），就像试验需要跑足够长时间排除季节影响

完整流程

1. 定义指标 — 主指标（CTR / 转化率 / 收入）+ 护栏指标（不能变差的指标，如加载时间）+ 反向指标

2. 计算样本量 — 需要确定：显著性水平 α（通常 0.05）、统计功效 1−β（通常 0.8）、基准指标值、最小可检测效应 MDE

最小样本量（每组）：n ≈ 2(z_α/2 + z_β)² × p(1−p) / δ² δ = MDE（最小可检测差异） 典型值：α=0.05, power=0.8 → z值分别为 1.96 和 0.84

3. 随机分流 — 用户级别（user_id hash）而非会话级别；确保随机性（AA测试验证）；避免 Simpson's Paradox

4. 运行实验 — 跑满至少一个完整周期（去除星期效应）；不要 peeking（中途看结果）；设置 guardrail alert

5. 分析结果 — Novelty effect / Primacy effect；Segmentation 分析（新老用户、设备类型）；网络效应；多重比较（Bonferroni 校正）

❌ A/B 测试最容易踩的 4 个坑
  • Peeking（偷看）：实验跑了一半发现 p=0.04 就停止上线——第一类错误率会从 5% 膨胀到 30%+。正确做法：提前计算好样本量，跑完再看；或用 Sequential Testing。
  • 用户级 vs 会话级分流：同一用户在不同会话看到不同版本，体验不一致且统计假设被破坏。应该用 user_id 哈希分流。
  • Novelty Effect（新奇效应）：新功能上线头两天用户因为好奇而多点击，之后回落。解决：等实验跑满至少 1-2 周后分析，并单独看"老用户"的结果。
  • 忽略网络效应（Network Effect）：适用社交类产品——给实验组用户推送通知，对照组也可能被影响（朋友给他们分享了）。需要用"地理隔离"或"时间隔离"来设计实验。

机器学习概览

🎯学完本节你会掌握：三类学习范式区别；偏差-方差权衡的诊断与处理；标准建模流程 CRISP-DM。
📚前置知识：统计学基础。学完后推荐：监督学习算法 → 无监督学习。
💡 直觉类比：偏差-方差 = 射箭的"准头"和"稳定性"
想象你在射箭，靶心是"真实规律"：
• 高偏差（欠拟合）= 瞄错方向：每次都射到靶子左边，箭很集中但就是不准（模型太简单，连训练集都没学好）
• 高方差（过拟合）= 手抖：有时射中有时偏很远，训练时很准但换个数据就乱了（模型记住了训练集的噪声）
• 最佳模型 = 准且稳：箭都集中���靶心附近
诊断口诀：训练误差大 → 高偏差（欠拟合），加复杂度；训练误差小但验证误差大 → 高方差（过拟合），加数据或正则化。

偏差—方差权衡

总误差 = Bias² + Variance + 不可约误差（噪声） 高偏差（Underfitting）：训练误差大，验证误差也大 → 模型太简单 高方差（Overfitting）：训练误差小，验证误差大 → 模型太复杂

降低偏差（欠拟合）

  • 增加模型复杂度
  • 添加更多特征
  • 减少正则化强度
  • 使用更强的模型

降低方差（过拟合）

  • 增加训练数据
  • L1/L2 正则化
  • Dropout / 剪枝
  • 集成方法（Bagging）
❌ ML 初学者最容易犯的错误
  • 先跑模型再做 EDA：跳过数据探索直接调模型，结果发现特征有严重泄漏或数据质量问题。正确顺序：EDA → 数据清洗 → 特征工程 → 建模。
  • 用测试集调参：把测试集看了，再优化模型，最终测试集得分虚高，上线后大幅下降。测试集只能用一次，超参调优应在验证集上进行。
  • 只看 Accuracy 就报告结果：数据不平衡时（如欺诈检测 0.1% 正例），预测全是负类也能得 99.9% Accuracy，但模型完全无用。永远要先看数据分布。

监督学习核心算法

🎯学完本节你会掌握：五类核心算法原理与适用场景；如何根据数据特征选择算法；XGBoost 关键调参技巧。
📚前置知识：ML 概览（偏差-方差）。学完后推荐：特征工程 → 模型评估。
💡 直觉类比：算法选择 = 选合适的工具，不是最贵的工具
• 逻辑回归 = 直尺：简单快速，用于线性可分的场景。解释性最强，适合"为什么这笔交易被判定为欺诈？"
• 决策树 = 流程图：把数据的规律变成一系列 If-Then 判断，人人能理解，但太"死记硬背"（过拟合）
• 随机森林 = 投票委员会：100 棵决策树各自学，最后投票。单棵树很脆弱，100 棵投票就稳定了（Bagging 降方差）
• XGBoost = 连续纠错：第一个模型犯错 → 第二个模型专门来纠正前者的错误 → 第三个纠正前两个的残差……（Boosting 降偏差）
• SVM = 找最宽的过道：在两类样本之间找最宽的"无人区"作为边界，支持向量是离边界最近的点

线性回归 & 正则化

损失：MSE = (1/n)Σ(yᵢ − ŷᵢ)² Ridge (L2)：+ λΣwⱼ² → 权重均匀收缩，保留所有特征 Lasso (L1)：+ λΣ|wⱼ| → 产生稀疏解，自动特征选择 ElasticNet：α·L1 + (1−α)·L2 → 兼顾两者

逻辑回归（分类）

P(y=1|x) = σ(wᵀx+b) = 1/(1+e^(−z)) 损失（交叉熵）：L = −[y·log(p) + (1−y)·log(1−p)] 注意：用交叉熵而非 MSE（MSE 对 sigmoid 输出梯度消失）
  • 多分类扩展：One-vs-Rest 或 Softmax（多项式逻辑回归）
  • 适合线性可分数据，训练快，可解释性强
  • 需要特征归一化，对异常值敏感

决策树

分类：Gini = 1 − Σpᵢ² （越小越纯，范围0~0.5） 信息增益 = H(parent) − Σ(nₖ/n)·H(childₖ) 回归：MSE 最小化分裂
  • 优点：可解释，不需归一化，处理混合类型
  • 缺点：不稳定（数据微小变化导致结构大改），容易过拟合
  • 控制过拟合：max_depth / min_samples_split / min_samples_leaf / 后剪枝(ccp_alpha)

随机森林 = Bagging + 特征随机化

  • 两个随机：① Bootstrap 抽样（有放回，约 63% 样本）② 分裂时随机选 √p 个特征
  • OOB 误差：约 37% 未被选中的样本作为验证集，无需额外交叉验证
  • 降低方差而非偏差，适合高方差（过拟合）的基学习器
  • 关键参数：n_estimators（越多越好，但收益递减）、max_features、max_depth

XGBoost = Gradient Boosting + 正则化 + 系统优化

串行训练：每棵树拟合前一轮的"伪残差"（负梯度方向） 目标函数：L = Σl(yᵢ,ŷᵢ) + Σ[γT + ½λ||w||²] （损失项 + 正则项：叶子数量T + 权重L2）
  • vs 随机森林：RF 并行降方差；XGB 串行降偏差；XGB 通常精度更高但更慢
  • 关键参数：learning_rate(0.01~0.1), max_depth(3~6), n_estimators, subsample, colsample_bytree, reg_alpha/lambda
  • 防过拟合：early_stopping_rounds + eval_set；subsample/colsample 引入随机性

支持向量机

  • 核心：最大间隔超平面，支持向量是最近的样本点
  • 软间隔参数 C：C 大→严格→低偏差高方差；C 小→宽松→高偏差低方差
  • 核函数：RBF 高斯核（γ大→局部→过拟合）；多项式核；线性核（高维稀疏首选）
  • 适用：文本分类（高维稀疏）、样本量不大时效果好；大数据集训练慢
❌ 算法选择的常见错误
  • "XGBoost 最好，所有问题都用它"：XGBoost 对高维稀疏数据（文本 one-hot）效果差；非常小数据集上 SVM 或逻辑回归可能更好；上线后可解释性要求高时，逻辑回归是更务实的选择。
  • 不做基线对比就上复杂模型：先跑逻辑回归基线。如果基线就有 0.92 AUC，XGBoost 调到 0.93 值不值得额外的复杂度？量化提升再决定。
  • XGBoost vs 随机森林搞混：随机森林 = 并行（各棵树独立训练，Bagging 降方差）；XGBoost = 串行（每棵树纠正前一棵的残差，Boosting 降偏差）。前者不容易过拟合，后者精度更高但需要调参防过拟合。

特征工程

🎯学完本节你会掌握：缺失值三种类型与处理策略；类别特征5种编码方式；时间特征周期性编码；特征选择方法；Data Leakage 识别与防范。
📚前置知识：监督学习算法。学完后推荐：模型评估 → Kaggle 项目实战。
⚠️业界经验：80% 的时间在数据和特征，20% 在模型。好特征 + 简单模型 往往胜过 差特征 + 复杂模型。
💡 直觉类比：特征工程 = 给考生"备考资料"，而不是换一个聪明的学生
模型就像一个学生，特征就是给他的复习材料：
• 缺失值处理 = 把残缺的笔记补全（不能留空白，模型看到 NaN 会出错）
• 编码 = 把文字（"北京"、"上海"）翻译成数字，因为模型只懂数学
• 时间特征的 sin/cos 编码 = 帮模型理解"23:59 和 00:01 其实很近"——如果直接用小时数字，模型会以为它们差 23 个单位
• Data Leakage（数据泄露） = 把"期末考试答案"混进复习材料——训练时成绩很好，但上线/考试时一败涂地

缺失值处理

类型说明处理方法
MCAR（完全随机缺失）缺失与任何变量无关均值/中位数/众数填充
MAR（随机缺失）缺失与其他已观测变量有关KNN填充、回归填充、mice多重填充
MNAR（非随机缺失）缺失与自身值有关（最难处理）添加缺失标记特征；业务逻辑填充

类别特征编码

  • One-Hot：低基数（<20类），无序类别；注意虚拟变量陷阱（drop first）
  • Label Encoding：有序类别（低/中/高）
  • Target Encoding：高基数（城市、商品ID）；用目标变量均值编码；需 Leave-One-Out 或加平滑防 data leakage
  • Frequency Encoding：用类别出现频率替换；保留分布信息
  • Embedding：类别数极大时（UserID）用神经网络学习稠密表示

时间特征（面试常考）

  • 年/月/日/小时/星期几/是否节假日/是否周末
  • 周期性编码：sin/cos 变换，避免 23:59 和 00:01 距离很远的问题
  • Lag 特征：前1/7/14/28天的值（时序预测必用）
  • 滑动窗口：过去 N 天均值/标准差/最大值
❌ 特征工程最危险的错误：Data Leakage（数据泄露）
  • Target Encoding 泄露：用整个训练集的目标均值编码某类别特征，包含了当前样本自身的目标值。正确做法：Leave-One-Out 或 K-Fold 内计算，或加平滑项。
  • 时序数据随机分割：用 random_split 分训练/验证集，导致"用未来数据预测过去"。时序数据必须按时间顺序切分，验证集永远在训练集之后。
  • 在全量数据上做 StandardScaler 再分割：fit_transform 整个数据集后再 train_test_split，测试集的统计信息泄漏到了训练过程。正确做法：只在训练集上 fit，然后 transform 测试集。
  • 用"手术结果"作为预测特征：预测患者是否会做手术，把"手术费用"作为特征——这是事后才知道的信息，线上预测时根本拿不到。

模型评估

🎯学完本节你会掌握：分类/回归各类指标含义与选择逻辑；Precision-Recall 权衡的业务含义；交叉验证方法。
📚前置知识：监督学习算法。面试关键：能根据业务场景解释为什么选某个指标。
💡 直觉类比：Precision vs Recall = "宁可错杀 vs 宁可放过"
场景1 — 垃圾邮件过滤（Precision 优先）：宁可让垃圾邮件漏进收件箱（漏报），也不能把重要邮件误判为垃圾（误报）。领导发给你的合同被过滤掉比收到几封垃圾邮件严重得多。→ 提高 Precision（减少误报）

场景2 — 癌症筛查（Recall 优先）：宁可多做几次活检（误报），也不能漏掉一个真正的患者（漏报）。漏掉一个癌症患者可能要命，多做一次活检只是浪费钱。→ 提高 Recall（减少漏报）

AUC-ROC 是什么？ 代表"随机取一个正例和一个负例，模型把正例排在负例前面的概率"。0.5 = 瞎猜，1.0 = 完美。AUC 不关心你设什么阈值，衡量的是模型的整体排序能力。
指标公式适用场景注意事项
AccuracyTP+TN / N类别均衡类别不平衡时无意义（99%预测负类也有99%准确率）
PrecisionTP/(TP+FP)误报代价高垃圾邮件：不想把正常邮件标为垃圾
RecallTP/(TP+FN)漏报代价高疾病筛查：不想漏掉患者

RMSE√(Σ(y-ŷ)²/n)回归对异常值敏感；量纲与目标一致
MAEΣ|y-ŷ|/n回归（鲁棒）不可微，对异常值不敏感
✅面试答题框架：当被问"用什么指标评估模型"时，先问业务场景（假阳性 vs 假阴性的代价），再选指标。例如：欺诈检测→高 Recall（宁可误报也不漏报）；推荐系统→NDCG/MAP@K；CTR预估→AUC-ROC。
❌ 模型评估的常见错误
  • 用 Accuracy 评估不平衡数据集：欺诈检测正例占 0.1%，全预测负类也有 99.9% Accuracy。面试官一问"你的模型 Recall 是多少"就漏了。任何正负比大于 1:5 的数据集都不能只看 Accuracy。
  • 混淆 AUC 好 = 概率准：AUC 衡量排序能力，但不保证预测的 0.8 概率真的意味着 80% 的发生率。推荐系统只需排序用 AUC；风控定价需要真实概率，还需做 Platt Scaling 或 Isotonic Regression 校准。
  • 用训练集 AUC 汇报结果：随便一个足够复杂的模型在训练集上都能达到接近 1.0 的 AUC。评估必须在训练时从未见过的验证集或测试集上进行。
练习题 #3 — 模型评估

超参数调优

🎯学完本节你会掌握：Grid Search / Random Search / Bayesian Optimization / Optuna 的原理与适用场景；如何避免超参调优中的 Data Leakage；Early Stopping 与 Pruning 加速技巧。
📚前置知识：监督学习算法、模型评估。为什么重要："好模型=正确算法+正确超参"，调参是 Kaggle 竞赛和工业部署中决定性的最后一步。
💡 直觉类比：超参调优 = 在黑暗的迷宫里找宝藏
超参数空间就像一个巨大的迷宫，你看不到地图，只知道当前位置的"得分"：
• Grid Search = 把每个房间都搜一遍：对于 3 个参数各 10 个值，要进 1000 个房间。房间太多，绝大多数时间花在"明显差"的角落
• Random Search = 随机跳着搜：虽然随机，但研究证明：大多数参数对结果影响很小，只有 2~3 个关键参数。随机搜可以在这些关键维度上采样更多不同的值
• Bayesian Optimization = 有记忆地搜：记住之前哪些区域得分高，下次优先探索"得分高的附近"（Exploitation）以及"还没探索过的角落"（Exploration）的平衡
• Early Stopping = 探索一半发现这条路不行就撤退：不浪费时间在"前几步就明显很差"的超参组合上
方法原理优点缺点适用场景
Grid Search穷举所有超参组合结果确定，覆盖全面组合爆炸（10参×10值=10¹⁰次）参数少（≤3个），范围明确
Random Search随机采样超参组合比 Grid 高效；等算力下找到更好结果（低维有效超参少）随机性，不保证最优参数多，初步探索
Bayesian Opt用代理模型（高斯过程/TPE）建模目标函数，选最有可能改进的点样本效率高，适合昂贵评估有状态，难并行；对高维支持差训练慢的深度模型
Optuna（TPE）Tree-structured Parzen Estimator；用历史好结果估计参数分布Python 原生，支持 Pruning，易集成需要定义 objective 函数工业界首选，通用
Hyperband/ASHA多臂老虎机 + Successive Halving：资源少时并行跑多组，提前杀掉差的极快，适合大规模并行需要训练过程可中断深度学习，Ray Tune
⚠️关键：超参调优必须在验证集上做，不能用测试集。否则超参会过拟合测试集（如 Kaggle 公榜 vs 私榜差距大）。正确流程：Train→CV 选超参→在 Train 上最优超参重新训练→Test 上最终评估（仅一次）。

Random Search 为什么优于 Grid Search？

Bergstra & Bengio（2012）的洞察： 大多数超参对性能的影响是低维的（只有少数超参真正重要） 假设有10个超参，其中只有2个重要： • Grid Search（每维5个值）：5¹⁰ ≈ 1000万次，但重要维度只采样5个点 • Random Search（1000次）：重要维度采样 ~1000 个不同点 结论：在"有效维度低"的情况下，Random Search 以更少次数覆盖更多有效点

Optuna Pruning（提前终止）

问题：某些超参组合在训练初期就明显很差，继续跑是浪费 解决：Pruner 在中间步骤评估，若当前试验明显差于历史最优则 Prune（终止） Optuna 的 MedianPruner：中间检查点的指标差于历史中位数 → Prune HyperbandPruner：结合 Successive Halving，更激进 等效加速：同等计算预算下，Pruning 通常能多探索 3~10 倍的超参组合

实战调参顺序（XGBoost 为例）

  • 第一步：固定 learning_rate=0.1，用 early stopping 确定 n_estimators（通常 100~1000）
  • 第二步：调树结构 max_depth（3~8）和 min_child_weight（1~10），控制过拟合
  • 第三步：调采样参数 subsample（0.6~1.0）和 colsample_bytree（0.6~1.0）
  • 第四步：调正则化 reg_alpha（L1）和 reg_lambda（L2）
  • 第五步：降低 learning_rate 到 0.01~0.05，相应增加 n_estimators（精细调整）
❌ 超参调优的常见错误
  • 用测试集选超参：CV 选超参、测试集只用一次。如果根据测试集结果反复调整参数，就等于把测试集当了验证集，最终结果虚高（Kaggle 公榜好但私榜差的根本原因）。
  • 先调 learning_rate：应先用 early stopping 确定 n_estimators，再调树结构，最后微调 lr。先调 lr 而不知道需要多少棵树，等于在沙上建楼。
  • 把 Grid Search 当"标准做法"：超参数多于 4 个时，Grid Search 是低效的。Random Search 用同等次数几乎必然找到更好的结果。

模型可解释性（SHAP / LIME）

🎯学完本节你会掌握：SHAP 值的博弈论原理；LIME 局部近似方法；全局 vs 局部解释的区别；何时用哪种方法；面试高频 Shapley Value 推导直觉。
📚前置知识：监督学习算法、模型评估。为什么重要：监管合规（GDPR 可解释要求）、模型调试、特征选择、业务决策落地。
💡 直觉类比：SHAP 值 = 公平分配"团队合作成果"给每个队员的贡献
想象一个销售团队，3 个人（特征）一起完成了 35% 的业绩（预测值）。基准是 50%（平均预测）。如何公平地算每个人的贡献？
Shapley 值的思路：枚举所有可能的"上场顺序"，算每个人在各种顺序中加入后带来的增量，取平均。
例如：张三（年收入）先上场能拿 20%，李四（信用分）先上场能拿 25%。但两人都在时，第二个上场的人贡献会缩小（因为第一个人已经覆盖了一些信息）。Shapley 值就是考虑了所有排列顺序后的"公平份额"。
• SHAP 值为正 → 该特征让预测值比基准高（如信用分高推高了贷款通过概率）
• SHAP 值为负 → 该特征让预测值比基准低（如高负债拉低了通过概率）
• 所有特征 SHAP 值之和 = 预测值 - 基准值（可加性，可以"对账"）

为什么需要可解释性？

黑盒模型（XGBoost/神经网络）精度高但不透明。可解释性工具回答三类问题：

  • 全局解释：哪些特征整体上最重要？（特征重要性排名）
  • 局部解释：这个具体预测是由哪些特征驱动的？（单样本解释）
  • 反事实：改变哪个特征可以翻转预测？（If-then 解释）

SHAP（SHapley Additive exPlanations）

来源：合作博弈论中的 Shapley 值 核心思想：特征 i 的 SHAP 值 = 它在所有可能的特征子集顺序中， 加入后带来的边际贡献的加权平均 ϕᵢ = Σ_{S⊆F\{i}} [|S|!(|F|-|S|-1)!/|F|!] · [f(S∪{i}) − f(S)] 性质（唯一满足以下4条公理的方法）： • 效率（Efficiency）：所有特征 SHAP 值之和 = 预测值 - 基线值 • 对称性（Symmetry）：等价特征的 SHAP 值相等 • 虚拟性（Dummy）：无贡献特征的 SHAP 值为 0 • 可加性（Linearity）：组合模型的 SHAP 值等于各模型 SHAP 值之和
SHAP 变体适用模型速度特点
TreeSHAPXGBoost/LightGBM/随机森林O(TLD²) 快精确值，工业界首选
DeepSHAP神经网络中等结合 DeepLIFT，近似值
KernelSHAP任意黑盒慢（采样）模型无关，适合对比研究
LinearSHAP线性模型极快精确，等价于线性系数×特征值

LIME（Local Interpretable Model-agnostic Explanations）

核心思想：在待解释样本 x₀ 附近，用可解释的简单模型（线性回归） 近似黑盒模型的局部行为 算法步骤： 1. 对 x₀ 进行扰动，生成邻域样本（对文本=随机 mask 词；对图像=超像素遮罩） 2. 用黑盒模型预测每个扰动样本的输出 3. 按照与 x₀ 的距离加权（距离越近权重越高） 4. 拟合加权线性回归：g(z) = wᵀz，最小化 Σ π(z)·(f(x)−g(z))² 5. 线性系数 w 即为特征的局部重要性

SHAP vs LIME 对比

SHAP

  • 有严格数学公理保证（唯一性）
  • 全局+局部均适用（beeswarm/waterfall/force图）
  • TreeSHAP 速度快，工业界主流
  • 可做特征选择（mean |SHAP|）
  • 缺点：特征相关性高时解释不稳定

LIME

  • 模型无关，支持文本/图像/表格
  • 直觉上易懂（局部线性近似）
  • 稳定性差（随机采样）
  • 邻域定义（带宽参数）敏感
  • 适合向非技术用户解释单个预测
❌ 可解释性工具使用的常见错误
  • 把 Feature Importance 等同于 SHAP：XGBoost 的 Feature Importance（gain/split count）是全局统计量，不考虑特征间的相互作用；SHAP 值是基于博弈论公理的严格分配，更可信。高 Feature Importance 的特征，SHAP 绝对值均值也高，但单个样本的 SHAP 值可能为负。
  • 用 SHAP 解释相关特征时过度自信：当特征高度相关（如"父亲收入"和"母亲收入"），SHAP 会把贡献随意分配给其中一个，另一个看起来不重要。相关特征的 SHAP 解释需要结合领域知识谨慎对待。
  • 全局 Feature Importance 覆盖个体差异："信用分是第一重要特征"是全局平均结论，对某个特定用户，"负债率"可能对他的预测起决定作用。局部 SHAP（Waterfall Plot）才能真正解释单个决策。

无监督学习

🎯学完本节你会掌握：K-Means / DBSCAN / GMM 原理与适用场景；PCA 降维数学直觉；t-SNE 可视化原理；聚类评估指标（Silhouette / DB Index）。
💡 直觉类比：聚类算法 = 不同的"分组方式"
想象把一堆地图上的地点分组成"商圈"：
• K-Means = 找 K 个商圈中心，每个地点归最近的中心：假设商圈是圆形的，需要事先说好有几个商圈（k）。北京三环内的密集点和郊区的稀疏点，K-Means 会以为郊区只是一个大商圈。
• DBSCAN = 找密集区域：足够密集的点自然成一个商圈，稀疏孤立的点被标记为"不属于任何商圈"（噪声点）。形状不规则的商圈也能识别，不需要事先说商圈数量。
• PCA = 把数据"压扁"到最重要的方向：就像把一个三维物体的影子投影到二维平面，选能保留最多信息（方差最大）的投影方向。
• t-SNE = 专门用来可视化的降维工具：高维的"近邻关系"在 2D 中也保持相近，但簇间距离没有意义，不能用于分析。

K-Means

算法步骤： 1. 随机初始化 k 个中心点（用 K-Means++ 改进初始化） 2. 分配：每个点归属最近中心点 3. 更新：重新计算每个簇的均值作为新中心 4. 重复 2-3 直到收敛（中心不再移动） 目标：最小化簇内平方误差 SSE = Σᵢ Σₓ∈Cᵢ ||x - μᵢ||²
  • 超参 k：Elbow Method（画 SSE vs k 曲线找肘部）；Silhouette Score 最大化
  • 局限：假设簇是球形；对异常值敏感；需要预先指定 k；不适合非凸形状
  • 变体：K-Medoids（更鲁棒）；Mini-Batch K-Means（大数据）；Bisecting K-Means

DBSCAN（密度聚类）

核心概念： ε（邻域半径）：点 p 的 ε-邻域内有 ≥ MinPts 个点 → p 是核心点 核心点：形成簇的中心；边界点：在核心点邻域内；噪声点：既非核心也非边界
  • 优点：无需指定 k；能发现任意形状的簇；自动识别噪声点（异常检测）
  • 缺点：高维数据效果差；不同密度簇效果差；ε 和 MinPts 难以调参
  • HDBSCAN：层次化版本，自适应密度，更鲁棒

高斯混合模型（GMM）

假设数据由 k 个高斯分布混合生成： p(x) = Σₖ πₖ · N(x | μₖ, Σₖ) 用 EM 算法（期望最大化）学习参数： E步：计算每个点属于每个高斯分布的后验概率（软分配） M步：用后验概率加权更新 μₖ, Σₖ, πₖ
  • vs K-Means：GMM 软分配（概率），K-Means 硬分配；GMM 允许椭圆形簇
  • 用 BIC/AIC 选 k：BIC = ln(n)·k − 2·ln(L̂)，越小越好

PCA（主成分分析）

目标：找到方差最大的投影方向 步骤： 1. 标准化数据（零均值单位方差） 2. 计算协方差矩阵 C = XᵀX / (n-1) 3. 特征分解：C = VΛVᵀ（V 是特征向量，Λ 是特征值） 4. 按特征值降序排列，取前 k 个主成分 解释方差比 = λᵢ / Σλⱼ；通常保留 95% 解释方差
  • 直觉：把数据旋转到方差最大的方向，丢弃低方差（噪声）维度
  • 注意：PCA 是线性方法，无法捕捉非线性结构；需先标准化；主成分不可解释
  • 用途：降维可视化；去除多重共线性；压缩特征（Whitening）

t-SNE（非线性降维可视化）

核心思想：高维中相近的点，低维中也应相近 高维：用高斯分布计算点对相似度 pᵢⱼ 低维：用 t 分布计算点对相似度 qᵢⱼ（t 分布尾部更重，防止拥挤） 最小化：KL(P||Q) = Σᵢⱼ pᵢⱼ log(pᵢⱼ/qᵢⱼ)
  • 超参 perplexity：平衡局部和全局结构，通常 5~50，需多次尝试
  • 注意：t-SNE 结果不可复现（随机初始化）；簇间距离无意义；不能用于新数据预测
  • UMAP：更快，保留更多全局结构，可用于新数据 transform

❌ 无监督学习的常见错误
  • K-Means 之前不做归一化：K-Means 使用欧氏距离，如果收入（1万~100万）和年龄（20~60）不归一化，收入会完全主导分群结果。必须先 StandardScaler 再聚类。
  • 用 t-SNE 分析簇间距离：t-SNE 只保留局部邻近关系，两个簇在 t-SNE 图上看起来很近，不代表它们在原始空间也很近。t-SNE 只用于"看有没有明显分群"的定性可视化。
  • 把 Elbow Method 当精确科学：Elbow Method 找的"肘部"通常很模糊，不同人看同一张图可能选不同的 k。应该结合 Silhouette Score 的数值和业务含义共同决定 k。

神经网络基础

🎯学完本节你会掌握：激活/损失/优化器选择逻辑；BN 训练推理的区别；梯度消失/爆炸的原因与解法；反向传播直觉。
📚前置知识：线性代数基础（矩阵乘法），ML 概览。学完后推荐：CNN → Transformer。
💡 直觉类比：神经网络 = 多级"信号处理流水线"
• 每层神经元 = 一组信号检测器：第一层检测边缘，第二层组合边缘成纹理，第三层组合纹理成形状……最后一层输出"这是猫的概率"
• 激活函数 = 神经元的"点火阈值"：ReLU 就是"低于 0 就沉默，高于 0 就原样传递"，避免了 Sigmoid 的"饱和区"问题（Sigmoid 在很大或很小的值时梯度接近 0，信号传不回去）
• 反向传播 = 错了就往前追责：预测错了 → 算每个参数"该为这个错误负多少责任"（梯度）→ 按比例修正参数。数学上就是链式法则：∂L/∂w₁ = ∂L/∂出口 × ∂出口/∂w₁
• Batch Normalization = 每层之间的"标准化检查站"：防止某一层的输出值越来越大或越来越小，导致后面的层难以训练

Batch Normalization 详解

对每个 mini-batch，每个特征维度： μ = mean(x_batch), σ² = var(x_batch) x̂ = (x − μ) / √(σ² + ε) ← 归一化 y = γ · x̂ + β ← 可学习缩放平移 训练时：用当前 batch 统计量 推理时：用训练时积累的移动平均（running_mean/running_var）
  • 解决 Internal Covariate Shift，允许更大学习率，加速收敛
  • 有轻微正则化效果（来自 batch 的随机性）
  • 序列/小 batch 场景用 Layer Norm；分组特征用 Group Norm

梯度消失

  • 深层梯度趋近于 0
  • Sigmoid/Tanh 激活导致
  • 解决：ReLU、残差连接（ResNet）、BN、LSTM

梯度爆炸

  • 梯度极大，训练不稳定
  • RNN 中常见（BPTT）
  • 解决：梯度裁剪（clip_grad_norm）、权重初始化
❌ 神经网络训练的常见错误
  • 分类任务用 MSE 损失：Sigmoid 输出 + MSE 损失组合会让梯度在接近 0/1 时趋近于 0（梯度消失）。分类任务必须用交叉熵（Binary/Categorical Cross-Entropy）。
  • BN 推理时用当前 batch 的统计量：训练时用 batch 均值/方差，推理时必须用训练过程中累积的移动平均（running_mean/running_var）。在 PyTorch 中要确保推理时调用 model.eval()。
  • 全零初始化权重：所有神经元初始值相同 → 梯度相同 → 每次更新相同 → 永远是"对称"的网络，等价于只有一个神经元。应用 He（ReLU）或 Xavier（Sigmoid/Tanh）初始化打破对称性。

卷积神经网络（CNN）

🎯学完本节你会掌握：卷积/池化操作原理；感受野计算；经典架构对比（VGG/ResNet/EfficientNet）；迁移学习策略；面试高频考点。
💡 直觉类比：卷积 = 用"放大镜"扫描图片
• 卷积核（Filter）= 一个放大镜：3×3 的放大镜在图片上滑动，每次看到一个 3×3 的小区域，判断"这里有没有我在找的模式（比如竖线/斜线/角点）"
• 多个卷积核 = 多种放大镜：64 个卷积核 = 64 个专门找不同图案的侦探，每人输出一张"热力图"（特征图），代表各自在图片各位置找到的证据强度
• 感受野 = 最终决策"看过"多大范围：浅层网络的感受野小（只看到局部纹理），深层网络感受野大（能看到整个物体的语义关系）
• ResNet 残差连接 = "短路线"：如果 F(x) 学不到有用信息，至少 x 直接传递过去，梯度回传时永远有一条高速公路（∂L/∂x = ∂L/∂y·(∂F/∂x + 1)），解决了深层网络梯度消失的根本问题

残差连接（ResNet 核心）

普通： y = F(x) 残差： y = F(x) + x ← skip connection（恒等映射） 梯度回传：∂L/∂x = ∂L/∂y · (∂F/∂x + 1) 关键：即使 ∂F/∂x 很小，梯度也至少为 ∂L/∂y，解决梯度消失

迁移学习策略

  • 特征提取（冻结全部）：新数据量极少（<1000张）；直接用预训练特征 + 新分类头
  • Fine-tuning 最后几层：中等数据量；解冻后几个 Block，用小学习率（1e-4 ~ 1e-5）
  • 全量 Fine-tuning：大数据量；先用小 lr 训练分类头，再解冻全网络
  • 学习率建议：分层学习率 — 浅层用 lr/10，深层用 lr，分类头用 lr
✅面试必答：为什么用预训练权重？ImageNet 特征（边缘/纹理/形状）具有通用性。即使目标域（医疗/卫星图像）与 ImageNet 差异大，预训练初始化也比随机初始化收敛更快、效果更好。
❌ CNN 和迁移学习的常见错误
  • Fine-tuning 时用太大的学习率：解冻预训练层后如果用和分类头一样大的 lr（如 1e-3），会破坏预训练权重学到的通用特征。应该用分层学习率：浅层 lr/100，深层 lr/10，分类头 lr。
  • 数据量太少却做全量 Fine-tuning：少于 1000 张图片时解冻全网络会严重过拟合。正确做法：数据少 → 冻结所有层只训练分类头；中等数据（1K~10K）→ 解冻最后几个 Block；大数据（>10K）→ 全量 Fine-tuning。
  • 忘记归一化到 ImageNet 统计量：使用预训练模型时，输入必须归一化到与预训练时相同的均值方差（mean=[0.485,0.456,0.406], std=[0.229,0.224,0.225]）。否则预训练权重对当前输入"认不出来"，收敛很慢甚至不收敛。
  • 把池化层和卷积层的参数量搞混：MaxPooling 层没有可学习参数（只是取最大值），参数量为 0。Conv2d(3,64,3×3) 的参数量 = 3×3×3×64+64 = 1792，不是 3×3×64 = 576。面试被问"参数量计算"时要区分。

Transformer

🎯学完本节你会掌握：Self-Attention 数学原理；Multi-Head 的意义；位置编码方案对比；Encoder/Decoder 架构适用场景。
📚前置知识：神经网络基础（BN/Dropout/残差），矩阵运算。学完后推荐：LLM 架构 → RAG → 微调。
💡"Attention is All You Need"（2017）— Transformer 是现代 NLP、CV 和 LLM 的统一骨架。
💡 直觉类比：Self-Attention = 开会时每个人"主动找相关的人对话"
想象一个10人会议，讨论"苹果公司的发展"：
• 普通 RNN = 传话游戏：消息从第1人传到第2人再到第3人……到第10人时第1人说的内容已经模糊了（长程依赖消失）
• Self-Attention = 直接对话：每个人（每个 token）同时问所有其他人"你和我说的话有多相关？"，把相关度高的人的发言加权组合起来作为自己的"更新后理解"。"苹果"这个词会同时注意到"公司"和"价值"，而不是只看左右邻居
• Q/K/V = 提问/标签/内容：每个人带着三样东西——Q（我想问什么？）K（我能回答什么话题？）V（我的实际内容是什么）。注意力分数 = 用 Q 去匹配所有人的 K，相关度高的人的 V 权重就大
• Multi-Head = 同时开多个议题的小组讨论：8个 head 就是8个并行讨论组，一组关注语法关系，一组关注语义相似，一组关注指代关系……最后把8组结论拼在一起

Self-Attention 机制

每个位置的 token 生成三个向量： Q（Query）= X·W_Q K（Key）= X·W_K V（Value）= X·W_V Attention(Q,K,V) = softmax(QKᵀ / √d_k) · V 直觉：每个词"问"(Q) 其他词是否与我相关 (K)，加权聚合信息 (V) √d_k 缩放：防止 d_k 大时点积过大→ softmax 饱和 → 梯度消失
  • Multi-Head Attention：h 个独立 head 并行，关注不同子空间，拼接后投影
  • 位置编码：sin/cos 函数（原始版）/ 可学习 PE / RoPE（旋转，现代 LLM 主流）
  • 计算复杂度：O(n²d)，长序列瓶颈；Flash Attention 用 IO-aware 算法优化
  • 残差连接 + LayerNorm：每个子层后都有，防梯度消失，稳定训练

Flash Attention — IO-Aware 注意力优化

💡标准 Attention 的瓶颈不在算力，而在显存带宽（HBM 读写）。Flash Attention 重新设计计算顺序，大幅减少 HBM 访问次数。
标准 Attention 的内存访问瓶颈： 1. 计算 S = QKᵀ → 写入 HBM：O(n²) 2. 计算 P = softmax(S) → 读写 HBM：O(n²) 3. 计算 O = PV → 读写 HBM：O(n²) 总计：3 次 O(n²) 的 HBM 读写 → n=16384 时约占满 A100 显存 Flash Attention v1/v2 核心思想（Tiling）： • 将 Q/K/V 分块加载到 SRAM（快速片上内存） • 在 SRAM 内完成 softmax + 加权求和（online softmax trick） • 只将最终结果 O 写回 HBM 一次 • HBM 访问量降至 O(n)，速度提升 2~4x，显存节省 10~20x
版本核心改进加速效果
Flash Attention v1Tiling + recomputation（反传时重计算 S，省显存）2~4× 加速，10× 省显存
Flash Attention v2更优的线程分配，减少 non-matmul FLOPs约 2× v1，接近理论峰值
Flash Attention v3利用 H100 Tensor Core 的异步流水线H100 上约 1.5× v2
Ring Attention多 GPU 切分序列，支持超长上下文（百万 token）线性扩展到任意序列长度

❌ Transformer 最容易混淆的 4 个概念
  • √d_k 缩放为什么必要：d_k=64 时，QKᵀ 的点积量级约为 O(8)，softmax 进入饱和区（输出接近 one-hot），梯度趋近 0。除以 8 将量级控制在 O(1)，softmax 保持"平滑"，梯度正常流动。不是"随便加的超参"。
  • Layer Norm 和 Batch Norm 用错场景：Transformer 用 Layer Norm（对每个样本的特征维度归一化），不用 Batch Norm（对 batch 维度归一化）。原因：NLP 中序列长度不一，batch 统计量不稳定；LLM 推理时 batch=1，BN 完全失效。
  • BERT（Encoder）和 GPT（Decoder）傻傻分不清：BERT 看到完整句子做理解任务（双向），GPT 只能看左边的词做生成任务（单向）。用 BERT 做生成 = 在考试时能看答案；用 GPT 做分类 = 只看了半道题。任务决定架构选择。
  • Attention 复杂度是 O(n²) 不是 O(n)：Self-Attention 中每个 token 要和所有其他 token 交互，序列长度翻倍 → 计算量翻4倍。1000 token 问题不大，100K token 就需要 Flash Attention / Sliding Window Attention 等优化。

LLM 大语言模型

🎯学完本节你会掌握：LLM 核心概念（KV Cache/Temperature/Hallucination）；RLHF vs DPO 对齐方案；Scaling Laws 含义。

💡 直觉类比：LLM 就是一个"极其博学的自动补全"
• 预训练 = 读完互联网上所有文字：把参数当作"压缩的世界知识"——70B 参数的模型把万亿 token 的知识蒸馏进去。不是"查数据库"，是把知识"记在脑子里"
• 自回归生成 = 每次只预测下一个词：给你"今天天气很"，模型输出"好"。再给"今天天气很好"，输出"，"……这就是 LLM 生成文本的方式——每步都是概率采样，不是确定性搜索
• Temperature = 创造力旋钮：T=0 就像"只说最可能的答案"（适合数学题）；T=1 是正常随机；T=2 像醉酒状态——答案天马行空。面试时要能解释"为什么代码生成用 T=0.2，创意写作用 T=0.9"
• KV Cache = 备忘录：对话中已经处理过的 token 的 K/V 矩阵存起来，下次生成新词时直接查备忘录，不用重新算——这就是为什么生成第2个词比第1000个词快的原因被消除了
• Hallucination = 自信地编造：模型本质是"概率分布"，它生成"看起来合理的文字序列"，不是"查证后的事实"。知识截止日期之后、罕见领域、具体数字——这些最容易出错

  • 更简单、更稳定，是 RLHF 替代方案
❌ LLM 使用和理解中的常见错误
  • 把 Temperature=0 当作"关闭随机性"：Temperature=0 是贪婪解码（每步取概率最大的 token），但模型本身是概率模型，不等于"确定性答案"。同一个问题问两次仍可能不同（因为 system prompt、tokenization 边界等因素）。要真正稳定结果，需加 seed 参数。
  • 认为更多上下文 = 更好理解：LLM 存在"Lost-in-the-Middle"现象：上下文窗口的中间部分容易被忽略，开头和结尾被关注最多。把最重要的信息放在 prompt 的最前面或最后面，不要埋在中间。
  • 把 LLM 的 Hallucination 当"bug"而非"feature"：Hallucination 是概率语言模型的内在特性，不是可以完全修复的 bug。缓解方案（RAG/Grounding/事实核查）能减少但不能消除。构建产品时需要假设 LLM 会出错，设计验证层。
  • Scaling Laws 理解错误：Scaling Law 说"更大模型+更多数据+更多算力→更好性能"，但 Chinchilla 定律更精确：给定算力预算，最优的做法是用较小模型+更多数据，而不是无脑堆参数。GPT-3（175B，300B tokens）其实训练不足。

提示工程

💡 直觉类比：提示工程 = 给一个聪明但字面理解的新员工写任务说明书
LLM 就像一个刚入职、极其聪明、但完全字面理解指令的实习生：
• 模糊指令 = 你让他"随便写个报告"：他真的会写个"随便"的报告——你不满意，但是你的指令确实不清楚
• Few-shot = 给例子比给说明更有效：与其写一段"报告格式规范"，不如直接给他看2份好报告——他会模仿格式、风格、结构
• Chain-of-Thought = 让他先打草稿："先列提纲，再写每个部分"比"直接给我最终结果"得到更好的输出——因为中间推理步骤帮助模型"理清思路"
• 角色扮演 = 激活特定知识域："你是一位 10 年经验的 DS 面试官"会让模型的回答风格、深度、措辞向面试官方向倾斜
• 负面约束 = 写"禁止事项"：不说"不要用专业术语"，模型仍会用。明确写出来，他才会遵守
  • 清晰指令：明确任务、格式、长度、受众、约束
  • Few-shot：提供 2~5 个示例引导模型，大幅提升一致性
  • 角色扮演："你是一位有 10 年经验的数据科学家…"
  • 结构化输出：要求 JSON / Markdown 表格 / 特定格式
  • 分步思考："请一步步分析" 触发推理能力
  • 负面约束："不要包含…" 明确排除不想要的内容
  • Zero-shot CoT：末尾加 "Let's think step by step." 即可触发
  • Few-shot CoT：示例中包含完整推理链，模型模仿
  • Tree of Thought（ToT）：多路径推理 + 评估 + 回溯，适合复杂问题
  • ReAct：推理（Reason）+ 行动（Act，调用工具）交替，Agent 的基础
  • Self-Consistency：多次采样取多数票，提升准确率 5~10%
  • HyDE：先让 LLM 生成假设答案，再用它检索，提升 RAG 召回质量
  • Program-Aided LM (PAL)：让 LLM 生成代码并执行，规避计算错误
  • Constitutional AI：让模型根据原则自我批评和修正
  • Prompt 注入防御：分隔符（XML tags）、系统提示优先级、输入清洗
❌ 提示工程中最常犯的错误
  • 靠感觉调 prompt，不做系统测试：改了一个词觉得"好像更好了"就认为改进有效——这是幸存者偏差。正确做法：构建评估集（20~50个样本），量化指标（准确率/相关度/格式符合率），把每次 prompt 改动当 A/B 测试。
  • Few-shot 示例质量差：示例如果包含错误、风格不一致、格式混乱，模型会学坏的一面。Few-shot 示例比 zero-shot 指令影响更大——垃圾示例比没示例更差。
  • Chain-of-Thought 在简单任务上浪费：CoT 对复杂推理（数学/逻辑）显著有效，对简单分类或提取任务反而可能引入噪声（模型推理出错误的中间步骤）。简单任务用清晰 zero-shot 即可。
  • 把 "Let's think step by step" 当万能咒语：Zero-shot CoT 对 GPT-4 级别的模型有效，但对较弱模型可能产生冗长但错误的推理链。要在目标模型上实测效果，而非盲目套用。

RAG 检索增强生成

💡 直觉类比：RAG = 给 LLM 配一个"实时图书馆员"
纯 LLM 的问题：知识被冻结在训练截止日期，且记忆容量有限（参数记不住所有文档细节）。RAG 的思路：
• 不让模型"记住所有书"，而是让它"用时查书"：把公司文档、最新政策、产品手册存入向量数据库，每次提问时先检索相关段落，再让 LLM 基于这些段落回答
• Embedding 向量 = 把文字翻译成坐标：把"机器学习"编码成 1536 维空间里的一个点，语义相近的文字在这个空间里的点也相近——这样就能用距离来衡量"相关程度"
• Chunk 大小 = 图书馆的"检索单位"：太小（50字）丢失上下文；太大（2000字）噪声多、检索不精准。通常 256~512 token 加 10% 重叠
• Hybrid 检索 = 结合"关键词搜索"和"语义搜索"：BM25 擅长精确词匹配（专有名词/数字），向量检索擅长语义理解，两者结合比任何一个单独都好

❌ RAG 系统最常见的 4 个坑
  • Chunk 太小导致上下文断裂：50~100 字的 chunk 检索精度高，但 LLM 拿到的段落缺乏上下文（一个段落开头没有主语）。解决：Parent-Document Retriever（检索小 chunk 定位，返回大 chunk 给 LLM）或加重叠（overlap=50~100 token）。
  • 只用语义检索，专有名词漏检：向量检索对语义相近词有效，但"GPT-4o"、"v3.2.1"、"SKU-8801"这类精确词，向量可能和其他词相似。必须加 BM25 关键词检索，用 RRF（Reciprocal Rank Fusion）合并排名。
  • "检索到了但 LLM 没用"：Context 传入 LLM 但答案仍然不准，通常是 prompt 格式问题：context 太长（超出有效注意力）、放在 prompt 中间（Lost-in-Middle）、格式混乱（LLM 不知道哪是文档哪是问题）。用 RAGAS 框架区分是"检索问题"还是"生成问题"。
  • 不做 RAG 评估就上线：拍脑袋觉得检索"好像还行"不够。需要评估 4 个维度：Context Precision（检索到的文档有多少是相关的）、Context Recall（相关文档有没有被检索到）、Answer Faithfulness（答案是否基于检索结果）、Answer Relevance（答案有没有回答问题）。

LLM 微调技术

💡 直觉类比：微调 = 在专业领域"再培训"一个通才
基础 LLM 就像一个读了很多书的通才大学生：
• Full Fine-tuning = 去读专业研究生：全面重塑知识体系，效果最好，但成本极高（需要 4×A100 训练 7B 模型，还可能灾难性遗忘原有能力）
• LoRA = 给通才配一个"领域专家助手"：核心能力（W）不变，旁路加了一个小模块（ΔW = AB）专门处理特定任务。A 矩阵（d×r）学"如何提炼输入"，B 矩阵（r×k）学"如何输出领域结果"，r 通常只有 4~16，参数量从百万降到几万
• QLoRA = 先把书压缩成 4-bit 漫画版（量化），再用 LoRA 训练：信息有损失（精度略降），但24GB 显卡能跑原本需要 160GB 的 70B 模型
• 什么时候微调 vs RAG：RAG 适合"需要实时查最新文档"；微调适合"需要改变输出风格/格式/行为模式"——比如让模型永远用中文回答、按特定格式输出 JSON。两者可以叠加使用
方法原理可训练参数适用场景
Full Fine-tuning更新全部参数100%数据充足，资源充裕，效果最好
LoRAΔW = AB（低秩分解），只训练 A/B0.1%~1%最常用，资源效率极高
QLoRA4-bit 量化模型 + LoRA极少消费级 GPU 微调 70B 模型

DPO直接偏好优化，无需奖励模型全部或部分对齐偏好，替代 RLHF
💡LoRA 原理：预训练权重 W ∈ ℝ^(d×k) 冻结，添加旁路 ΔW = A·B（A: d×r, B: r×k，r≪min(d,k)）。前向传播：h = Wx + ΔWx。只有 A 和 B 参与梯度更新，参数量从 d×k 降到 r×(d+k)。
❌ LLM 微调的常见错误
  • 数据太少就做 Full Fine-tuning：少于 1000 条样本做全量微调会严重过拟合，且可能引发灾难性遗忘（Catastrophic Forgetting）——原有的通用能力下降。少量数据首选 LoRA（r=4~8），数据充足再考虑全量。
  • LoRA rank 设太大：r=256 不等于效果更好——更高的 rank 增加过拟合风险，训练更慢。工业界主流是 r=4~16，对大多数任务已足够。从小 rank 开始，如果欠拟合再增大。
  • 不做 RLHF/DPO 对齐就上线：SFT（监督微调）让模型学会"做任务"，但不保证输出无害、有帮助、诚实。直接把 SFT 模型对外开放，会有安全风险。对齐（RLHF 或 DPO）是从"能做"到"应该做"的关键步骤。
  • 混淆微调和提示工程的适用边界：如果 few-shot prompting 已经能达到 90% 的效果，微调带来的 2% 提升是否值得额外的标注成本和工程维护？先试提示工程，再决定是否微调。

时间序列分析

🎯学完本节你会掌握：平稳性检验；ACF/PACF 解读；ARIMA 建模流程；ML 方法（LightGBM 时序特征）；预测评估指标。前置：统计学基础。
💡 直觉类比：时间序列 = 病历记录，预测 = 诊断未来健康状况
• 平稳性 = 病人"状态稳定"：均值/方差不随时间变化。发烧发到一半分析无意义——先让病人"退烧"（差分）到平稳状态，才能建模
• ACF（自相关）= "今天的体温和过去N天有多相关"：ACF 在 lag=7 还很高，说明一周前的数据对今天有预测力（周期性）
• ARIMA 的直觉：AR（Auto-Regressive）= 用自己的历史值预测（今天销量 = 昨天×0.6 + 前天×0.3）；MA（Moving Average）= 用过去预测误差来修正（今天预测值 = 均值 + 昨天误差×0.4）
• LightGBM 时序特征 = 把"时序关系"翻译成"表格特征"：lag_7 = 上周同期值，rolling_mean_28 = 过去28天均值。这样把时序问题转成普通表格机器学习问题——关键是绝对不能用未来数据作为特征（Data Leakage）
• MAPE 的坑：销量为 0 时，MAPE = 无穷大。零销量商品（长尾品类）必须用 SMAPE 或 WRMSSE 等其他指标

核心概念

概念定义检验/工具
平稳性均值、方差、自协方差不随时间变化ADF 检验（p<0.05 → 平稳）；KPSS 检验
自相关 ACF序列与其自身 lag k 版本的相关系数拖尾 → MA 项；截尾 → AR 阶数
偏自相关 PACF排除中间 lag 影响后的纯相关截尾 → AR 阶数；拖尾 → MA 项

趋势长期上升或下降一阶差分去趋势；对数变换去指数增长

ARIMA 建模流程

ARIMA(p, d, q)： p = AR 阶数（用 PACF 截尾确定） d = 差分次数（使序列平稳，ADF 检验） q = MA 阶数（用 ACF 截尾确定） SARIMA(p,d,q)(P,D,Q)[s]：加入季节性成分，s = 季节周期 选模型：先 ADF 检验 → 差分使其平稳 → 看 ACF/PACF → 拟合 → 用 AIC/BIC 选最优

ML 方法 vs 传统方法

统计方法（ARIMA/Prophet）

  • 可解释性强，有置信区间
  • 适合单序列，数据量少
  • Prophet 自动处理节假日/缺失值
  • 假设较强，难以捕捉复杂非线性

❌ 时间序列分析最容易踩的坑
  • 用 random_split 而不是 time_split：随机分割会让"未来数据"进入训练集（比如用2024年12月的数据预测同年1月），训练时AUC虚高，上线后大幅下滑。时序数据必须按时间顺序切分——验证集永远在训练集之后。用 sklearn 的 TimeSeriesSplit。
  • Lag 特征构建泄露未来信息：计算 lag_1 时，如果训练集的 label 已经是"下一天的值"，lag_1 实际上用到了当天的真实值——这就是 Data Leakage。需要仔细对齐特征和标签的时间戳。
  • ARIMA 建模之前不检验平稳性：直接对非平稳序列（有趋势/季节性）拟合 ARIMA(p,0,q)，参数估计会失效。必须先做 ADF 检验（p值>0.05→非平稳），差分到平稳后再确定 p, q。
  • 把 MAPE 用在零值序列上：长尾商品（偶尔才有销量）的真实值为 0 时，MAPE = |0-ŷ|/0 = ∞，整个评估失效。这类场景必须换用 SMAPE、MAE 或定制指标。

Apache Spark 基础

💡 直觉类比：Spark = 把一个大任务拆给一百个人同时做
• 分布式计算 = 流水线工厂：数据被切成 N 个 Partition，分配给 N 个 Executor 并行处理，最后汇总结果——1TB 数据 100 台机器各处理 10GB，比 1 台机器快 100 倍
• Lazy Evaluation = "先收集所有任务，优化后再一起执行"：df.filter().groupBy().sort() 这三步不会立即执行，只是记录在 DAG 里。直到 df.count() 或 df.write() 时，Catalyst 优化器看完整 DAG 后选最优执行计划——就像司机先看完整个路线再决定走哪条路，而不是每到十字路口才想
• Shuffle = 工厂里最贵的"跨部门协调"：groupBy/join 需要把相同 key 的数据集中到同一个节点，这涉及网络传输（序列化→传输→反序列化），是 Spark 任务最慢的环节。一个 Shuffle 可能让速度下降 10x
• Cache = 把中间结果"贴墙上"而不是每次重新计算：迭代算法（ML 训练）不 cache 中间 DataFrame，每次迭代都要重读磁盘，10次迭代 = 读10次原始数据
概念说明
RDD弹性分布式数据集，不可变，分布式计算基础（低级 API）
DataFrame有 Schema 的分布式表，Catalyst 优化器，推荐使用
Lazy EvaluationTransformation 不立即执行，Action（collect/count/write）触发 DAG 执行
ShufflegroupBy/join 触发跨节点数据重新分布，最昂贵操作（网络 I/O）
Partition并行度单位；理想每个 partition 128MB；AQE 可自动调整
Cachedf.cache()：MEMORY_AND_DISK；迭代计算时必用，避免重复读取
❌ Spark 初学者最常犯的错误
  • 对大表用 collect()：df.collect() 把整个 DataFrame 拉到 Driver 节点内存——1TB 的表直接 OOM（内存溢出）。查看数据用 df.show(20) 或 df.limit(100).toPandas()，不要 collect 大表。
  • Transformation 以为立即执行：df2 = df.filter(col > 0) 执行完立刻检查 df2 的行数——但这行代码只是"定义了一个过滤操作"，还没真正执行。直到 df2.count() 才触发。所以 df.filter() 后面没有 Action，这行代码等于什么都没做。
  • 忘记 schema，用 inferSchema=True 读大文件：inferSchema 需要 Spark 扫描整个文件才能推断类型，对 TB 级文件极慢。生产环境应手动定义 schema，速度快 10x。
  • 重复计算没有 cache：df.groupBy().agg() 结果被后续代码用了 3 次，但没有 df.cache()，Spark 会重新计算 3 次。把被多次引用的 DataFrame cache 住是基本优化习惯。

Spark 性能优化

⚠️Spark 面试最高频考点：数据倾斜的识别与处理，以及 Broadcast Join 的使用场景。
💡 直觉类比：数据倾斜 = 一个人做了所有活，其他人干等着
假设10个工人按"城市"分配处理快递包裹，北京包裹占总量60%：
• 9个工人处理其他城市，很快完成；1个工人处理北京，苦苦挣扎30分钟——整个任务被这1人卡住
• Salting（加盐）= 把北京包裹再按序号分成10份：北京_0、北京_1……北京_9，每个工人只处理北京的 1/10，负载均衡了；合并时再按原始 key 汇总
• Broadcast Join = 把小表复印10份，每人桌上放一份：这样根本不需要"按城市集中数据"（Shuffle），每个工人自己查自己桌上的小表即可——彻底消除 Shuffle
• repartition vs coalesce：repartition = 重新洗牌分配（有 Shuffle，适合增加分区）；coalesce = 直接合并相邻分区（无 Shuffle，只能减少分区）
数据倾斜

Salting 方案

给热点 key 加随机后缀（0~N），将一个大 partition 打散成 N 个小 partition；join 时对方表扩展 N 倍

Join 优化

Broadcast Join

小表（<10MB）用 broadcast hint，避免 shuffle；spark.sql.autoBroadcastJoinThreshold 默认 10MB

文件格式

Parquet + 分区

列式存储，比 CSV 快 10x+；按高频过滤列分区（如 dt=2024-01）启用 Partition Pruning

AQE

自适应查询执行

Spark 3.0+，运行时动态调整分区数、处理倾斜、选择 Join 策略；默认开启

分区操作： repartition(n) → 全 shuffle，均匀分布，增加/调整分区 coalesce(n) → 无 shuffle，只减少分区（合并），更高效 最佳分区大小：128MB，公式：total_data_size / 128MB
❌ Spark 性能调优的常见错误
  • 把 coalesce 当 repartition 用，反而更慢：coalesce(1) 把所有数据合并到一个 partition——后续的并行度变为 1，整个 DAG 串行执行。如果是为了"写出一个文件"，用 coalesce(1) 后再 write；但如果是为了"增加并行度"，必须用 repartition。
  • Broadcast Join 阈值设太大：把 autoBroadcastJoinThreshold 设到 1GB，"小表"需要广播到每个 Executor——100 个 Executor 就复制了 100GB 数据到内存，反而 OOM。Broadcast Join 适合真正的小表（通常 <100MB）。
  • 不看 Spark UI 就调参：每个性能问题都需要先在 Spark UI 定位（Stage 视图看慢 Task，SQL 视图看 Join 策略，Storage 视图看 Cache 命中），而不是凭感觉加机器。加机器解决不了数据倾斜问题。
  • Salting 之后忘记对方表 explode：对热点 key 加随机后缀（user_id + "_" + random(0,9)），但对方表没有对应扩展，join 的 key 对不上——全部变成空结果。两边都需要处理。

A. 直接 COUNT(DISTINCT active_users) / COUNT(DISTINCT registered_users)

❌ SQL 窗口函数最常见的 4 个错误
  • ROW_NUMBER / RANK / DENSE_RANK 搞混：三者并列时行为不同——ROW_NUMBER 无并列强制唯一（1,2,3,4）；RANK 有并列但跳号（1,1,3,4）；DENSE_RANK 有并列不跳号（1,1,2,3）。面试常考"取每组 Top 1 保留并列"→用 RANK 或 DENSE_RANK，不能用 ROW_NUMBER
  • 忘记 PARTITION BY 导致全局排名：漏写 PARTITION BY 时窗口函数对全表排名——ROW_NUMBER() OVER(ORDER BY score) 是全局第几，不是组内第几。找"��个用户的第一次购买"必须加 PARTITION BY user_id

  • 连续天数问题忘记先去重：同一用户同一天可能有多条记录（多次登录），直接用 ROW_NUMBER 减日期会得到错误分组。应先 SELECT DISTINCT user_id, login_date，再做日期 − ROW_NUMBER 技巧

因果推断

🎯学完本节你会掌握：相关 vs 因果的本质区别；三大因果推断方法（DID / PSM / IV）；潜在结果框架；大厂 DS 面试高频的"观测数据下的因果估计"问题。
📚前置知识：统计学基础（假设检验）、A/B 测试。为什么重要：现实中无法做完美 RCT 时，因果推断帮助从观测数据中估计"如果我们做了某件事，结果会如何"。
⚠️核心认知：相关 ≠ 因果。冰淇淋销量和溺水率正相关（共同原因：夏天）。如果你无法随机分配处理（Treatment），直接比较处理组和对照组会有选择偏差（Selection Bias）。
💡 直觉类比：因果推断 = 在"无法做实验"时，尽量还原出"如果做了实验"的结论

最理想的因果估计是 随机对照实验（RCT）——随机分组保证处理组和对照组"除了处理之外完全一样"。但现实中往往无法随机：不能随机给人发病、不能随机关闭某城市的 Uber。因果推断的目的就是用统计技巧，从观测数据中"模拟"出近似随机实验的效果。

  • DID（双重差分） = "时光机对比法"：若没有干预，两组趋势本该相同——干预前后的差异之差就是因果效应
  • PSM（倾向得分匹配） = "找一个和你一模一样但没参与实验的人"：控制所有可观测特征后配对对照组
  • IV（工具变量） = "找间接影响"：找到一个只通过处理影响结果的"外生冲击"，绕过未观测混淆变量
  • RDD（断点回归） = "利用规则门槛"：阈值两侧的人几乎相同，唯一差别是是否跨过分界线
  • 选择偏差是核心敌人：处理组和对照组"本来就不一样"，直接比均值的结果 = ATT + 偏差

潜在结果框架（Rubin Causal Model）

对每个个体 i，存在两个潜在结果： Y₁ᵢ：接受处理（Treatment=1）时的结果 Y₀ᵢ：未接受处理（Treatment=0）时的结果 个体因果效应：τᵢ = Y₁ᵢ − Y₀ᵢ ← 基本问题：永远只能观测到一个 平均处理效应 ATE = E[Y₁ − Y₀]（全人群） 处理组的平均处理效应 ATT = E[Y₁ − Y₀ | T=1]（更常关注） 选择偏差：E[Y|T=1] − E[Y|T=0] = ATT + 选择偏差项 → 选择偏差 = 处理组和对照组"本来就不一样"的部分

DID — Difference-in-Differences（双重差分）

适用场景：政策/干预前后有面板数据，有处理组和对照组 核心假设：平行趋势假设（Parallel Trends） → 若没有干预，处理组和对照组的趋势应相同 估计量：DID = (Ȳ_treat,post − Ȳ_treat,pre) − (Ȳ_control,post − Ȳ_control,pre) = [处理组后−前] − [对照组后−前] 直觉：双重差分消除了： 1. 处理组自身的时间趋势 2. 两组之间的固定差异（不随时间变化的个体效应）
  • 经典案例：Card & Krueger（1994）研究最低工资法对就业的影响；Uber 进入某城市对出租车收入的影响
  • 验证平行趋势：画处理组和对照组干预前的趋势图；Event Study 检验
  • Staggered DID：不同单位在不同时间点接受处理，需用 Callaway-Sant'Anna 或 Sun-Abraham 估计量
  • 回归形式：Y = α + β₁·Treat + β₂·Post + β₃·(Treat×Post) + ε，β₃ 即 DID 估计量

PSM — Propensity Score Matching（倾向得分匹配）

适用场景：无法随机化，但有丰富的协变量（混淆变量）数据 倾向得分 e(X) = P(T=1 | X)：在给定特征 X 的条件下，被分到处理组的概率 步骤： 1. 用 Logistic 回归估计倾向得分 e(X) 2. 将处理组样本与倾向得分相近的对照组样本匹配（1:1 or 1:k） 3. 检查匹配后两组的协变量平衡（SMD < 0.1） 4. 在匹配后的样本上估计 ATT 核心假设（CIA - Conditional Independence Assumption）： Y₁, Y₀ ⊥ T | X → 条件独立：控制 X 后，处理分配与潜在结果无关
  • 匹配方法：最近邻匹配 / 核匹配 / 半径匹配 / Mahalanobis 距离匹配
  • 局限：只能控制已观测变量，对未观测混淆变量无效；匹配后样本量减少
  • 更优方案：IPW（逆概率加权）直接用倾向得分加权，保留全部样本

IV — Instrumental Variables（工具变量）

适用场景：存在未观测混淆变量，或处理变量内生性问题 工具变量 Z 需满足3个条件： 1. 相关性（Relevance）：Z 与处理 T 强相关（弱工具变量问题：F > 10） 2. 外生性（Exogeneity）：Z 与误差项 ε 不相关（不受未观测混淆影响） 3. 排他性（Exclusion Restriction）：Z 只通过 T 影响 Y，无直接路径 两阶段最小二乘（2SLS）： 第一阶段：T̂ = α + βZ + γX + u → 提取 Z 引发的 T 变动 第二阶段：Y = α' + δT̂ + γ'X + ε → δ = LATE（局部平均处理效应）
  • 经典工具变量：出生季度→教育年限（Angrist & Krueger）；距医院距离→就医概率；彩票随机分配→移民决策
  • LATE vs ATE：IV 估计的是 Compliers 的处理效应（工具变量改变了处理状态的那些人），不是全人群
  • 大厂应用：自然实验设计（如算法随机 bug 作为工具变量）、随机推送作为工具变量估计展示→购买的因果效应

RDD — Regression Discontinuity Design（断点回归）

适用场景：处理分配由连续变量的某个阈值决定（准自然实验） 核心思想：阈值 c 两侧的样本除处理状态外，其他特征几乎相同 → 比较阈值两侧的 Y 差异即为因果效应 Sharp RDD（硬断点）： 处理 T = 1{X ≥ c}（完全由阈值决定） 估计：E[Y|X=c⁺] − E[Y|X=c⁻]（局部平均处理效应） Fuzzy RDD（软断点）： 阈值改变处理概率，但非完全决定（类似 IV 的二阶段思路）
  • 经典案例：信用评分 660 分门槛→贷款利率；考试分数线→奖学金获得→收入；年龄门槛→医疗保险
  • 识别假设：样本不能精确控制运行变量 X（没有人能精确卡在阈值附近）；其他协变量在阈值两侧连续
  • 带宽选择：IK/CCT 最优带宽；带宽越窄估计越可信但方差越大
  • 可视化验证：画 X 的密度直方图（McCrary 检验：阈值处不应有密度跳跃）
❌ 因果推断最容易混淆的 4 个概念
  • 相关 ≠ 因果，控制变量 ≠ 消除偏差：回归里加入更多变量无法完全消除选择偏差——若处理分配本身是内生的（如活跃用户更可能收到推送），OLS 估计量仍有偏。需要随机化或准自然实验才能识别因果
  • DID 的平行趋势假设可能不成立：如果处理组在干预前趋势就和对照组不同，DID 估计量是有偏的。必须事先画趋势图验证；Staggered DID（不同时间点处理）用普通 DID 公式会严重偏误，需用 Callaway-Sant'Anna 估计量
  • PSM 只控制可观测混淆变量：PSM 的核心假设（CIA）是"控制 X 后，处理分配与潜在结果独立"——但若存在未观测混淆（如个人动机、能力），PSM 仍有偏。LaLonde（1986）论文已经证明 PSM 无法复现随机实验结论
  • IV 估计的是 LATE，不是 ATE：IV 只估计"因工具变量影响而改变处理状态的那些人（Compliers）"的处理效应——不是全人群平均效应。弱工具变量（First Stage F < 10）会导致 IV 估计量严重偏倚

练习题 — 因果推断
某产品经理想评估"推送通知"对用户次日留存率的因果影响。直接比较收到推送和没收到推送的用户留存率，这样做有什么问题？最合适的方法是？
A. 没问题，样本量足够大就行
B. 存在选择偏差（活跃用户更可能收到推送也更可能留存）；应做随机实验（A/B测试）或用工具变量法
C. 用逻辑回归控制所有变量就能消除偏差
D. 用 DID 双重差分，比较开通推送前后的留存率
核心问题是选择偏差：推送通常只发给有一定活跃度的用户，这些用户本来留存率就高（即使不发推送也会留存）。直接比较会高估推送效果。正确方法：① 随机 A/B 测试（金标准）；② 若已有观测数据，用随机分配的"是否有资格收推送"作为工具变量估计 LATE；③ PSM（倾向得分匹配）控制用户活跃度等混淆变量，但无法控制未观测变量。DID 需要面板数据且处理前有对照，此场景不直接适用。

进阶 DID PSM
🔗 数据集：Kaggle — LaLonde Job Training Dataset 📁 格式：CSV，经典因果推断 benchmark 数据集，445 处理组 + 对照组，包含收入前后 🎯 任务：估计职业培训项目对就业收入的因果效应（ATT）
DID 双重差分 PSM 倾向得分匹配 平行趋势检验 协变量平衡 (SMD) ATT 估计 敏感性分析
  • 简单对比（有偏）：直接比较处理组与对照组的 1978 年收入差值 → $1794；但处理组本来基线收入就低（选择偏差）
  • DID 估计：(处理组78年−处理组75年) − (对照组78年−对照组75年)；验证平行趋势假设（画 75~76 年收入趋势图）
  • PSM 实现：Logistic 回归估计倾向得分 P(T=1|年龄/教育/种族/婚姻)；最近邻 1:1 匹配；检验匹配后 SMD < 0.1
  • 对比三种方法：Naive(有偏) vs DID vs PSM vs 真实实验值（$1794）；分析各方法的偏差来源

面试题 #6 — 概率推理
抛一枚硬币100次，正面60次，如何判断这枚硬币是否公平？
A. 60% > 50%，所以不公平
B. 用二项分布的比例 z 检验，计算 p-value，看是否拒绝 H₀: p=0.5
C. 再抛100次看第二次结果
D. 贝叶斯方法，必须提供先验
H₀: p=0.5，H₁: p≠0.5（双尾）。Z = (p̂−p₀)/√(p₀(1−p₀)/n) = (0.6−0.5)/√(0.25/100) = 2.0。p-value ≈ 0.046 < 0.05，拒绝 H₀，有统计显著性（但 Cohen's h ≈ 0.2，效果较小，实际影响有限）。贝叶斯方法也可行，但标准答案是频率学派假设检验。
面试题 #7 — ML 系统设计
设计一个电商 CTR 预估系统，你会怎么做？
A. 定义问题→特征工程（用户/商品/上下文）→模型（LR→Wide&Deep→DIN）→离线评估→在线 A/B→监控
B. 直接上 GPT-4，效果最好
C. 用随机森林，可解释性最强
D. 收集数据，训练模型，部署上线，完成
CTR 系统设计要点：① 特征：用户侧（历史CTR/类目偏好/活跃度）+ 物品侧（标题embedding/历史CTR/价格）+ 上下文（时间/设备/位置）+ 交叉特征；② 模型演进：LR（可解释）→ Wide&Deep（记忆+泛化）→ DIN（注意力权重）→ DIN-BERT；③ 关键挑战：特征工程线上线下一致性（training-serving skew）；④ 离线评估：AUC/GAUC（按用户分组）；⑤ 监控：预测分布偏移、特征缺失率、线上CTR。

模型演进详解：
• LR：线性模型，速度极快，需手动构造特征交叉，可解释性最强
• Wide & Deep（Google 2016）：Wide部分=LR（记忆：直接记住高频共现模式，如"炸鸡+啤酒"）+ Deep部分=DNN（泛化：学习从未见过的特征组合）
• DIN（Deep Interest Network，阿里 2018）：核心创新是注意力机制：用户历史行为序列中，和当前候选商品相关的行为权重更高。传统模型对历史行为取平均pooling（所有历史行为等权重），DIN改为：Attention(候选商品, 历史行为) → 加权求和，让模型关注"最近买过同类商品"等强相关行为。
• DIEN（阿里 2019）：在DIN基础上用GRU捕捉兴趣演化（时序信息），而非独立对待每个历史行为

面试题 #9 — 特征工程
